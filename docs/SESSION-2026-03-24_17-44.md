## Key Decisions Made

1. **One Graph, Three Interfaces**: Behavior cards, Logic Editor, and voice all edit the same LogicGraph. Cards are computed views, not separate storage.

2. **Logic Editor for ALL objects**: Not just buttons. Any object can be a source node in the logic graph.

3. **Container via parentId**: Flat object list preserved. Parent-child is a metadata pointer, not nested data. Drag-into-container detection on pointerUp.

4. **Block type routing in engine**: `evaluateBlock()` switches on `blockType`, keeping each evaluator isolated. New types are one case statement.

5. **Listeners via snapshot comparison**: JSON-serialize watchable props every render, compare to previous. Simple, works with 20-object cap.

6. **Behaviors are projections**: `graphToBehaviors()` pattern-matches the graph each render. Zero sync drift. Complex patterns just don't generate a card.

7. **Component Library = saved group + logic**: Not a new abstraction. Save a container's objects + wiring as JSON fragment, load and ungroup to customize. Planned for Phase 2C.

8. **Don't refine mid-plan**: Phone tests are for bugs only. Deep dive on connections happens after Phase 5 (which is now).

## Architecture Snapshot

```
Canvas (12 object types)
  ├── shape, image, button, text, input, timer
  ├── slider, toggle, progress, dropdown, counter
  └── container (parent-child via parentId)

LogicGraph (single source of truth)
  ├── nodes: LogicBlock[] (7 types)
  ├── listeners: ListenerNode[] (3 trigger types)
  ├── wires: Wire[]
  └── variables: Record<string, any>

Three Interfaces (same graph)
  ├── Behavior Cards (WHEN/THEN dropdowns, presets)
  ├── Logic Editor (node graph, wires, color-coded blocks)
  └── Voice Commands (~90 functions)

Execution Pipeline
  Source event → executeFromSource() → trace wires →
    → evaluateBlock() per block type →
    → updateObject() on targets →
    → propagate (depth-limited)

Listener Pipeline
  useEffect on objects change → snapshot compare →
    → fire matching listeners → executeFromSource()
```
