## Key Bugs Found & Fixed During Testing

| Bug | Root Cause | Fix |
|-----|-----------|-----|
| `listeners.length` undefined | Persisted Zustand state missing new fields | Guard all access with `\|\| []` + storage key bump to v2 |
| Double undo per action | Store + handler both pushing snapshots | SKIP_SNAPSHOT list for store-managed operations |
| Text can't be dragged | Tap-on-selected immediately entered edit | Changed to double-tap + pencil icon only |
| Tutorial cuts off at Logic Editor | `open_logic_editor` opens overlay, hides runner | Removed editor open/close from recipes — wiring works without UI |
| Calculator shows 1 not 55 | Math block used button toggle (1/0) as operand | Added `mathLeft` ValueRef so math reads from input objects |
| Button colors invisible | Transparent gradient on dark background | Solid HSL gradients at 40%/30% lightness |
| Recipe applies properties to wrong object | `handleFunction` used stale `selectedRef` | Reads fresh state from `useLabStore.getState()` on every call |
| Stale browser cache | Next.js serving old compiled JS | Deleted `.next/` cache, fresh build |

## Key Discoveries

1. **Math blocks needed `mathLeft`** — the original design only had `inputValue` as left operand, which works for simple chains but not for "read two independent inputs." Building the calculator revealed this immediately.

2. **`handleFunction` must read fresh state** — using refs that update via useEffect creates a race condition during recipe execution where multiple function calls happen faster than React renders.

3. **Tutorials can't open the Logic Editor** — the full-screen overlay blocks the tutorial runner. All wiring commands work without the editor UI open, which is actually better for tutorials (less visual noise).

4. **Button gradients need solid colors** — transparent overlays on a dark canvas are invisible. This was a design bug carried from the original prototype that nobody caught because buttons were always tested in isolation.
